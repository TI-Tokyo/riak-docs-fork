&gt;&gt;&gt; &gt;&gt; Maps and Sets are stored just like a regular Riak object, but using a
&gt;&gt;&gt; &gt;&gt; particular data structure and object serialization format. As you have

&gt;&gt;&gt; these

&gt;&gt;&gt; &gt;&gt;
&gt;&gt;&gt; &gt;&gt; It is possible to write a MapReduce map function (in Erlang) which
&gt;&gt;&gt; &gt;&gt; retrieves a provided object by type/bucket/id and returns the size of
&gt;&gt;&gt; it's
&gt;&gt;&gt; &gt;&gt; data. Would such a thing be of use?
&gt;&gt;&gt; &gt;&gt;
&gt;&gt;&gt; &gt;&gt; It would not be hard to write such a module, and I might even have
&gt;&gt;&gt; some
&gt;&gt;&gt; &gt;&gt; code for doing so if you are interested. There are also reasonably
&gt;&gt;&gt; good
&gt;&gt;&gt; &gt;&gt; examples in our documentation -
&gt;&gt;&gt; &gt;&gt; http://docs.basho.com/riak/latest/dev/advanced/mapreduce
&gt;&gt;&gt; &gt;&gt;
&gt;&gt;&gt; &gt;&gt; I haven't looked at the Python PB API in a while, but I'm reasonably
&gt;&gt;&gt; &gt;&gt; certain it supports the invocation of MapReduce jobs.
