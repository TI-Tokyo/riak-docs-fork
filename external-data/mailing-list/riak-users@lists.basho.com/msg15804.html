Riak Recap - Feb 27, 2015

Matthew Brender Fri, 27 Feb 2015 09:27:31 -0800 

Hi Riakators,

Here's your regular Recap of what's been happening in the last two three
weeks.
<https://gist.github.com/mjbrender/a44f743bc9e5ee668b05#fresh-release>Fresh
release

   - Riak 2.0.5 <http://docs.basho.com/riak/latest/downloads/> is out (full
   release notes
   <http://docs.basho.com/riak/latest/community/release-notes/>)
      - This release includes the fix for RIAK-1482
      
<https://github.com/basho/riak/blob/2.0/RELEASE-NOTES.md#riak-1482---maps-and-sets-incompatibility-in-204>,
      introduced in Riak 2.0.4 and made Map and Set datatypes
unreadable between
      upgrades
      - You can also export admin commands that use clique as CSV, which is
      handy (RIAK-1431
      <https://github.com/basho/riak/blob/2.0/RELEASE-NOTES.md#changes>)
   - Riak CS 1.5.4
   
<http://lists.basho.com/pipermail/riak-users_lists.basho.com/2015-February/016769.html>
was
   also released recently and has release notes
   
<http://docs.basho.com/riakcs/latest/cookbooks/Riak-CS-Release-Notes/#Riak-CS-1-5-4>
well
   worth a read

<https://gist.github.com/mjbrender/a44f743bc9e5ee668b05#other-code>Other
code

   - The Elixir client <https://github.com/drewkerrigan/riak-elixir-client> now
   includes connection pooling! Thanks Drew Kerrigan!
   - While still in POC, Jon Glick has a native OS X App for Riak
   <https://github.com/glickbot/riak-app> in the works
   - We're glad to see a project called nkbase
   <https://github.com/Nekso/nkbase> is leveraging riak_core

<https://gist.github.com/mjbrender/a44f743bc9e5ee668b05#recently-answered>Recently
answered

   - Ryan helps remind us to use the Riak Erlang/OTP
   <http://docs.basho.com/riak/latest/ops/building/installing/erlang/> when
   building Riak from source
   - Michael Martin resolve his own Erlang client issue from earlier this
   month and I captured it on Stackoverflow here
   
<http://stackoverflow.com/questions/28766962/trouble-with-put-get-in-riak-testing>
   - Shawn helps us recall that connections to Riak are dropped on exception
   
<http://lists.basho.com/pipermail/riak-users_lists.basho.com/2015-February/016701.html>
and
   gives a helping hand
   - And much more on this month's threads
   
<http://lists.basho.com/pipermail/riak-users_lists.basho.com/2015-February/thread.html>

<https://gist.github.com/mjbrender/a44f743bc9e5ee668b05#still-open>Still
open

   - Cosmin has an interesting edge case that loses an exception in the
   Java Client
   
<http://lists.basho.com/pipermail/riak-users_lists.basho.com/2015-February/016673.html>
and
   he'd like others to weigh in
   - Chris has an interesting set of errors
   
<http://lists.basho.com/pipermail/riak-users_lists.basho.com/2015-February/016780.html>
that's
   unanswered
   - Shawn found that ACLs are not set correctly for riak-cs
   
<http://lists.basho.com/pipermail/riak-users_lists.basho.com/2015-February/016826.html>
which
   may need more investigation
   - Matt Brooks has a very thoughtful question he calls Using Riak for
   Data with many Entities and Relationships
   
<http://lists.basho.com/pipermail/riak-users_lists.basho.com/2015-February/016848.html>
   - Jason has a repair-2i failure erroring with index_scan_timeout
   
<http://lists.basho.com/pipermail/riak-users_lists.basho.com/2015-February/016847.html>
